## 📌 Project Overview

This project implements a binary-class object detection system trained to distinguish **shoplifting** activity from **normal** customer behavior in retail CCTV footage. The pipeline covers the full ML lifecycle — from dataset acquisition and model fine-tuning to ONNX export and frame-by-frame video annotation — making it suitable for integration into real-world retail surveillance systems.

**Problem Statement:**  
Retail shrinkage due to shoplifting costs the global industry billions annually. Traditional surveillance relies on human monitoring, which is error-prone and unscalable. This project automates theft detection using computer vision, enabling real-time alerting without manual oversight.

## 📊 Model Performance

Metrics reported on the **validation split** after 20 training epochs:

| Metric | Score | Interpretation |
|--------|-------|----------------|
| **Precision (P)** | 0.5570 | ~56% of detections are true positives |
| **Recall (R)** | 0.8747 | Catches ~87% of actual shoplifting events |
| **mAP@50** | 0.5648 | Accuracy at 50% IoU overlap threshold |
| **mAP@50-95** | 0.4760 | Mean accuracy across strict IoU thresholds |

> **Design Note:** The model is intentionally tuned toward **high recall** over precision. In a loss-prevention context, missing a theft event (false negative) is more costly than generating a false alarm (false positive) that a human operator can quickly dismiss.

## 🔁 ONNX Export & Runtime Inference

The trained model is exported to ONNX format for framework-agnostic deployment:

```python
model.export(
    format="onnx",
    simplify=True,   # Reduces graph complexity via onnx-simplifier
    dynamic=True,    # Enables variable batch size at inference
    nms=True         # Bakes Non-Maximum Suppression into the graph
)
```

**ONNX Runtime inference example:**

```python
import onnxruntime as ort
import numpy as np

session = ort.InferenceSession("best.onnx", providers=["CPUExecutionProvider"])
input_name = session.get_inputs()[0].name

# Feed a preprocessed frame: shape (1, 3, 640, 640), dtype float32, normalized [0, 1]
frame_tensor = np.random.randn(1, 3, 640, 640).astype(np.float32)
outputs = session.run(None, {input_name: frame_tensor})
print("Output shape:", outputs[0].shape)
```

**Supported execution providers:** `CPUExecutionProvider`, `CUDAExecutionProvider`, `TensorrtExecutionProvider`

## 🎬 Video Inference Pipeline

The notebook includes a full OpenCV-based video processing loop:

1. Load video with `cv2.VideoCapture`
2. Extract frame metadata (width, height, FPS)
3. Initialize `cv2.VideoWriter` for output
4. Per-frame: run `model.predict()` → extract bounding boxes → draw overlays
5. Annotate each detection with: **RED bounding box** + **ALERT label** + **confidence %**
6. Write annotated frame to output video
7. Auto-download completed video

**Label format on-screen:**
```
ALERT: shoplifting (87.3%)
```

## ⚠️ Known Limitations

- **Precision gap:** mAP@50 of 0.56 indicates room for improvement. Increasing training epochs and dataset size would likely close this gap.
- **Dataset dependency:** Performance is bounded by the quality and diversity of the Roboflow dataset used. Real-world generalization requires more varied training data across lighting conditions, camera angles, and store layouts.
- **Video inference requires manual upload:** The notebook expects a video file at a hardcoded path — update `input_video_path` before running.
- **No tracker:** Detections are per-frame; the same individual may generate multiple independent alerts without identity continuity across frames.
